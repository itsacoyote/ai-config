---
name: branch-names
description: Use when creating a new branch to start any new work.
---

Branch names should be short but still descriptive of the work to be implemented. Use the conventional commits naming scheme for types to prepend the name of the branch.

NEVER create a branch name that uses only the type. For example, never create a branch called `feat` or `chore`.

## Branch Names

```
<type>/<short-slug>
```

Examples: `feat/agent-model-assignments`, `fix/token-refresh`, `chore/remove-dead-code`
